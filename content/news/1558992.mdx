当前的“正在安装 Windows 11”界面主要出现在用户通过 Media Creation Tool 或 ISO 镜像中的 setup.exe 重新安装系统时。这一界面本身并不算难看，也很少直接影响用户体验，但它在视觉风格上并未与 Windows 11 的整体设计方向完全统一。此前微软虽然多次强调会持续打磨 Windows 11 的外观，但外界普遍认为，这类低频出现的界面不会成为短期优先项。然而，微软设计总监 March 在社交平台 X 上发文表示，这一安装界面已被列入他们的设计优化清单，只是由于“极少被看到”，暂时被其他 UI 打磨项目排在了前面，但团队“终究会处理掉它”。

从表面上看，这只是一个并不起眼的界面更新，但从微软过往的态度来看，却具有一定信号意义。长期以来，微软对底层组件乃至各种不常见系统界面的视觉一致性并不算投入太多精力，更谈不上“逐个清理”。如今微软开始把视线延伸到安装程序这类低频入口，说明其对统一 Windows 11 全局体验的态度正在发生变化，设计团队希望连“角落里的界面”也纳入整体规范。

尽管 Windows 11 的视觉效果相较早期版本已有明显提升，但系统中仍散落着大量“遗留界面”。在某些场景中，用户仍会遇到 Windows 8 时代的界面，例如锁屏或设置中的部分页面。如果深入到控制面板等旧式管理入口，还可能跳出风格明显属于 Windows 3.1 时代的对话框，给人一种“时空错乱”的观感。文章指出，只要控制面板及其相关旧式对话框仍承担大量高级功能，微软在设计统一路上的前进就难言彻底。

微软此前已经表示，会先着手解决 Windows 设置（Settings）应用中的多处设计问题，并在此基础上逐步替代控制面板。这不仅是界面风格的统一问题，也牵涉到操作逻辑和使用效率：目前在很多情况下，用户在控制面板中完成某项操作所需点击次数，明显少于在设置应用中完成同一任务。微软清楚这一痛点，因此在未显著优化设置应用体验之前，不敢贸然砍掉控制面板。

在功能层面，控制面板中仍承载着大量与网络、打印机相关的旧式驱动和配置选项，其中不少尚未迁移到设置应用。微软设计负责人在 X 上表示，他们在迁移过程中必须格外谨慎，因为稍有不慎就可能导致多个功能失效，尤其会对企业用户以及依赖老旧终端或协议的开发者造成影响。一旦在这些关键场景中出现兼容性问题，微软不仅会在口碑上承受压力，也可能影响到企业级部署的稳定性和升级意愿。

正因如此，微软在统一设计的推进路径上采取了“先修好新界面，再考虑退场旧界面”的策略。具体做法包括：优化设置应用的信息层级和交互流程，将控制面板中的关键配置逐步抽取并重构到设置应用中，同时保持尽可能完善的兼容性测试。在这一思路下，从安装界面到锁屏、从设置到底层对话框，都是未来设计团队要逐一“收尾”的对象。

文章作者认为，微软这一次在设计上的表态与实际动作，或许代表着 Windows 11 将真正朝着“统一且现代”的视觉与交互方向迈进一步。如果微软能在不“惹恼”高阶用户和企业客户的前提下，完成控制面板等遗留组件的功能迁移，再配合对零散旧界面的清理，Windows 11 的整体观感与使用连贯性将有望获得质的改善。作者最后也向读者抛出问题：你最希望微软优先修正 Windows 11 中哪一部分界面？希望通过收集反馈，将这些诉求转达给微软团队。

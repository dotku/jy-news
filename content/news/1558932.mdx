在现版 Windows 11 中，“开始”菜单自推出以来一直存在争议。部分用户认为其在高分辨率屏幕上显得过大，如果不调整缩放设置，观感并不理想。 更受诟病的是新引入的“分类”布局：系统会自动为应用归类，但用户不仅无法自由编辑分类，还很难根据自己的使用习惯进行整理，导致界面在视觉和使用上都显得臃肿。

此前，媒体曾曝出微软在确定当前开始菜单方案之前，内部至少测试过五套完全不同的设计，并有多张从未公开的原型界面截图流出。 这些原型展示了不同的信息层级、应用呈现方式和布局思路，但最终都没有被采纳，仅停留在实验阶段。 在这波内部设计稿被分享至社交媒体后，引发了用户和设计从业者的广泛讨论，也促使微软设计团队更加正面回应外界关切。

在最新的回应中，March Rogers 在 X 上表示，新的开始菜单设计背后有一系列用户研究和反馈支撑，目标是让用户更轻松地访问自己真正需要的应用。 他同时透露，微软当前已经在开发额外的自定义选项，不过尚未公布具体功能形态和上线时间。 从其表述来看，微软并不会推翻现有开始菜单重新大改，而是更可能在现有框架上增加可调节项目，为用户提供更高自由度。

目前，开始菜单在自定义层面给用户的空间非常有限，尤其是在“分类”布局上线之后，这一点更为突出。 有关“分类”功能的吐槽也在微软反馈平台上长期占据高赞位置，一些用户明确要求能自行定义分类，认为没有编辑能力的自动归类只会增加界面噪音和使用负担。 还有用户直言，该设计如果经过有经验的用户界面专家把关，本不应该以当前形态进入正式版本。

从实现机制上看，Windows 11 的开始菜单分类并未使用云端或人工智能，而是依赖本地的一个约 15MB 的 JSON 文件来完成自动归类。 该文件预设了诸如“娱乐”“社交”“创意”等类别，并通过应用的包家族名称与这些类别进行匹配。 但由于描述信息不完整或元数据不规范，一些应用无法被准确识别，只能被归入“其他”类别，加剧了体验割裂感。

这也是为什么外界普遍推测，一旦微软真正推出新的开始菜单自定义功能，“分类”相关设置很可能是改动的重点之一。 如果用户能够手动调整分类、合并或重命名分组，甚至完全关闭自动分类机制，开始菜单的实用性和可控性都有机会得到显著提升。

除了界面和自定义层面，开始菜单在底层技术栈上也在发生变化。微软近期已经确认，正在将开始菜单从基于 React Native 的实现迁移到 WinUI，这一调整旨在降低延迟、提升整体性能和稳定性。 对用户而言，这意味着打开开始菜单、调用搜索和加载应用列表等操作的响应速度有望进一步加快。

有消息人士透露，微软内部还在评估更多交互层面的改进方案。 其中一项正在讨论中的变化，是允许用户在小尺寸和大尺寸之间手动调整开始菜单界面的整体大小，而无需依赖系统的全局缩放设置。 不过，相关功能仍处于规划阶段，目前尚无明确的发布时间表，微软也没有给出将通过哪一批次更新向用户推送这些改进。

在经历了数轮设计试验和长期的用户争议之后，Windows 11 的开始菜单正站在又一次调整的节点上。 对微软而言，如何在统一设计语言、性能优化和用户高度多样化需求之间取得平衡，将直接影响这一核心入口在未来数年的形态。 对大量依赖开始菜单工作和娱乐的用户来说，这轮改动能否真正让他们“掌控自己的开始菜单”，也将成为观察 Windows 11 体验演进的重要指标。

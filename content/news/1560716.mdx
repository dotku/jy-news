根据此次更新内容，Zed 1.0 已加入对 DeepSeek-V4-Pro 和 DeepSeek-V4-Flash 的支持，这两款模型也是近期刚刚推出的新模型。与此同时，新版本还带来了可在不同会话间保留的书签功能，并进一步增强了与 Git 命令行的集成能力，继续向更完整的现代开发工具形态靠拢。

除了 AI 与开发流程相关功能的扩展，Zed 1.0 还着重改善了跨平台一致性。报道指出，相较于此前以 macOS 为主的体验，这次更新让 Windows 和 Linux 用户获得了更接近的功能与稳定性表现，其中包括修复 Windows 平台 Alt-Tab 切换后的焦点异常问题，以及 Linux 平台 X11 环境下空格键输入故障等问题。

对于使用 Vim 模式的用户，新版本同样进行了针对性优化。此前在移动操作时出现的界面闪烁，以及在可视化选择后粘贴时可能覆盖剪贴板内容的问题，都已在这次更新中得到修复，进一步提升了编辑过程中的流畅性和可靠性。

不过，Zed 1.0 并非只有新增功能，也包含一项可能影响部分用户使用习惯的变更。开发团队移除了 preferred_line_length 选项，转而采用 bounded soft wrapping 方案；这一调整属于破坏性更新，可能会对部分依赖特定显示偏好的用户造成不便。

从背景来看，Zed 由三位曾参与 GitHub 旗下 Atom 文本编辑器开发的贡献者打造。Atom 已在 2022 年停止维护，GitHub 当时将重心转向 Visual Studio Code；而 Zed 在 2024 年开源之后，作为一款采用 Rust 编写的代码编辑器，已获得红杉资本 3200 万美元融资。虽然编辑器本体可以免费使用，但部分 AI 功能则采用付费方式提供。

对于已经安装 Zed 的用户，可通过左上角菜单进入更新检查。尚未安装的用户，则可以前往 Zed 官方网站下载体验这一 1.0 正式版本：

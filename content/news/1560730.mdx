Servo 新近加入了多项网页平台能力支持，包括带有 “range” 类型的 HTML input 元素、SVG 标签中的 width 与 height 属性，以及若干新的 CSS 特性和 DOM API。这表明 Servo 仍在持续补齐现代浏览器所需的基础能力，并稳步推进对主流 Web 标准的实现。

与此同时，Servo 的演示浏览器 Servo Shell 也进行了命名调整。项目方表示，这一程序现在安装后的二进制名称改为 servoshell，在 Windows 平台上则为 servoshell.exe，不再像此前那样统一使用 servo 这一名称。

在平台兼容性方面，Servo 团队也在继续推进对 FreeBSD 的支持。报道称，当前发布的 Servo 0.1 版本仍然存在一些 FreeBSD 构建问题，而在最新开发代码中，虽然已有所推进，但仍有少数遗留问题需要进一步解决；相关细节目前已通过 GitHub 议题页面对外公开。

除了 FreeBSD 适配工作之外，Servo 开发者还在继续增强该引擎的嵌入式友好性，希望让它对应用开发者而言，成为 Chromium Embedded Framework 之外更具可行性的替代方案。与此同时，项目也在推进与 Crates.io 新发布机制相关的工作，为其 Rust 生态下的发布与分发提供支持。

整体来看，这份最新进展说明 Servo 目前仍在围绕三条主线同步推进：一是补充 Web 平台功能，二是改善跨平台构建与 FreeBSD 支持，三是提升嵌入式使用场景下的可行性。更多有关这些改进的具体内容，已由 Servo 官方博客在最新日志中作进一步说明：

https://servo.org/blog/2026/04/30/march-in-servo/

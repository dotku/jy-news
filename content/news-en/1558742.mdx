This 1.0 official release brings several improvements in stability and reliability, including the introduction of the NdisWdfGetAdapterContextFromAdapterHandle interface to enhance the reliability performance of future versions on the Windows network stack. Simultaneously, version 1.0 has perfected the notification mechanism for MTU changes, achieving "correct" MTU change notifications, which helps maintain connection quality and performance under different network environments and link conditions.

Officials stated that this release also includes a large number of bug fixes, further solidifying WireGuard's production usability foundation on the Windows platform.

For those interested in learning technical details or participating in testing, you can read the v1.0 official release announcement via the project mailing list to obtain more complete change logs and future development directions:

https://lists.zx2c4.com/pipermail/wireguard/2026-April/009580.html
